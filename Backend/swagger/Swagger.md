### **Swagger (OpenAPI) – Notes**

**Definition:**  
Swagger is an open-source framework used to design, build, document, and test **RESTful APIs**.  
It is now part of the **OpenAPI Specification (OAS)** standard.

---

### **Main Tools in Swagger**

1. **Swagger Editor**
    
    - Used to design APIs using **YAML** or **JSON** syntax.
        
    - Helps define endpoints, parameters, and responses clearly.
        
2. **Swagger UI**
    
    - Automatically generates **interactive API documentation** from the API definition.
        
    - Allows developers to **test API endpoints** directly from the browser using “Try it out”.
        
3. **Swagger Codegen (or OpenAPI Generator)**
    
    - Automatically generates **client SDKs** and **server stubs** in different programming languages.
        
    - Saves time and ensures consistent code structure.
        

---

### **Why Swagger is Useful**

- Makes API documentation **interactive and easy to understand**.
    
- Ensures **clear communication** between backend and frontend developers.
    
- Allows **quick testing** of endpoints.
    
- Supports **automatic code generation** for faster development.
